Core Concept

If you understand Docker,
you understand AgentCraft

同样的分层思维，同样的声明式定义，同样的 CLI / Daemon 架构。只是这次管理的不是容器，而是 AI Agent。

AgentCraft

Docker AgentCraft 本质
Dockerfile Agent Template (JSON) 声明式定义"这个 Agent 是什么"
Image Template Registry 已验证、可复用、可分发的定义
Container Agent Instance 从定义实例化的运行单元，拥有隔离文件系统
docker run agent create + start 创建实例并启动
Volume Domain Context Skills / Prompts / MCP 物化为持久化文件
Writable Layer Memory Layer 运行时积累的可写状态 — 后天经验

docker commit Memory Promotion 实例经验提升为可共享知识
Network ACP / MCP 实例间及外部的通信通道
docker-compose Workflow + SubAgents 多实例编排
Vision

Agents that learn, share,
and embed into your workflow

Agent 不应该每次启动都失忆。让 Agent 进化、共享知识、成为工作流的一部分。

L3

Template Evolution

种群进化 — 高置信度经验经人工审核写回 Template

L2

Shared Memory

跨实例共享 — 验证过的经验提升到共享知识池

L1

Instance Memory

实例记忆 — 每次 session 结束自动提取经验

L0

Session

会话 — Agent 在 context window 内工作

Agent 会进化

实例在多次会话间积累经验 — 错误模式、用户偏好、最佳实践。下次启动时记忆自动注入，不再重复犯同样的错误。

Template 不变 ≠ Agent 不变。DNA 一生不变，但人一直在学习。进化发生在 Memory Layer。

Agent 会共享

当经验在多个实例中被验证，自动提升到共享知识池。实例 A 踩过的坑，实例 B 不再重蹈覆辙。

像 docker commit 将容器改动固化为镜像，Memory Promotion 将实例经验固化为团队知识。

Agent 嵌入工作流

One-Shot 在 CI 中自动执行后销毁。Service 作为虚拟雇员接入 IM。ACP Proxy 让外部应用调用 Agent。MCP 让 Agent 互相协作。

Agent 不是孤岛，是工作流中可插拔的节点 — 像容器在编排系统中一样。
See It In Action

Manage Agents like containers

熟悉的命令模式，熟悉的工作流。

agentcraft — terminal
# 启动 Daemon ≈ systemctl start docker
$ agentcraft daemon start
✓ Daemon started (pid: 42851)
# 查看可用模板 ≈ docker images
$ agentcraft template list
code-review-agent   Code review + security audit
bilibili-analyzer   Video content analysis
# 从模板创建 Agent ≈ docker run
$ agentcraft agent create reviewer --template code-review-agent
✓ Agent created: reviewer
Skills → AGENTS.md  |  Prompts → system.md  |  MCP → mcp.json
# 与 Agent 交互 ≈ docker exec
$ agentcraft agent run reviewer --prompt "Review error handling in src/"
Analyzing... Found 3 issues in error handling.
# 停止时自动提取经验 ≈ docker stop (+ commit)
$ agentcraft agent stop reviewer
✓ Memory extracted: 2 error patterns, 1 best practice
# 再次启动时带着记忆 — 不再失忆
$ agentcraft agent start reviewer
✓ Re-materialized with 3 memory insights
How It Works

Define → Create → Run → Interact → Evolve

1

Define

用 JSON Template 声明 Skills、Prompts、MCP 工具

2

Create

物化 Template 为隔离的 Instance Workspace

3

Run

选择 LaunchMode 启动，Daemon 管理生命周期

4

Interact

CLI / ACP 发送任务，Agent 在 workspace 内工作

5

Evolve

Session 结束提取经验，下次启动注入 Memory

Roadmap

Building the full stack

像 Docker 生态一步步成长一样，AgentCraft 有序演进。

Phase 1 Done

核心运行时 — 进程管理、LaunchMode、崩溃重启

≈ containerd + runc

Phase 2 Done

Agent 拼装与交互 — Domain Context 物化、CLI 对话

≈ docker build + exec

Phase 3 Active

通信与协议 — ACP Proxy ✓ + MCP Server

≈ docker network

Phase 4 Planned

扩展体系 — Plugin 架构、Heartbeat、Scheduler

≈ docker-compose

Phase 5 Planned

记忆系统 — Memory Layer、共享知识池、进化

≈ union fs + commit

Phase 6 Vision

ACP-Fleet — Daemon → ACP Server、Fleet 标准化

≈ docker swarm
