# Skill Skeleton

用于统一 AgentCraft skill 的最小流程骨架。新建或改写 skill 时，优先采用以下章节顺序：

1. 角色定义
2. 触发条件或指令解析
3. 前置检查
4. 执行流程
5. 升级规则
6. 完成状态
7. 禁止行为
8. 输出格式

推荐约束：

- 前置检查只写会改变执行安全或结果可信度的条件
- 升级规则要明确何时停止自主推进并报告用户
- 完成状态至少包含 `DONE`、`BLOCKED`、`PARTIAL`
- 禁止行为只写高风险动作，不写常识性废话
- 输出格式优先固定字段，减少每次自由发挥

默认判定标准：

- `DONE`: 已完成目标动作，且验证或收口动作已执行
- `PARTIAL`: 完成了部分目标，但仍有未消除的不确定项
- `BLOCKED`: 缺少关键上下文、权限、依赖或被安全规则阻断

除非 skill 的职责天然允许直接修改，否则默认遵循：

- 先收集证据，再决定动作
- 先说明停止条件，再说明自动推进路径
- 发现跨边界风险时中止，不隐式扩 scope
