# Frontend Guidelines

> Frontend guidance is intentionally minimal in the current phase.

## Current Status

Actant is not using frontend documentation as a primary architecture truth source right now.  
The current repository baseline is defined by:

1. `.trellis/spec/index.md`
2. `docs/design/contextfs-architecture.md`
3. `docs/design/actant-vfs-reference-architecture.md`
4. `docs/planning/contextfs-roadmap.md`

## Rule For UI Work

If you touch UI, dashboard, site, or wiki content:

- do not reintroduce old product narratives
- do not treat historical CLI-first or dashboard-roadmap text as authoritative
- align wording and information model with ContextFS

## Practical Constraint

View-layer work is currently secondary.  
If a frontend document is missing, outdated, or ambiguous, prefer updating the core spec/design docs first and keep UI changes minimal.
